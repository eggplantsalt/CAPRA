# 06 常见报错与故障排查

> 本文列举实际运行中最常见的问题及解决方案，按场景分类。

---

## 1. 环境安装问题

### 1.1 `ModuleNotFoundError: No module named 'libero'`

**原因：** LIBERO 仿真环境未安装。

```bash
pip install libero
# 如果 pip 版本较旧：
pip install --upgrade pip
pip install libero
```

### 1.2 `ModuleNotFoundError: No module named 'draccus'`

```bash
pip install draccus
```

### 1.3 `ImportError: cannot import name 'OpenVLAForActionPrediction'`

**原因：** 项目未以 editable 模式安装。

```bash
cd /path/to/openvla-oft
pip install -e .
```

### 1.4 MuJoCo 相关错误

```
Error: GLFW initialization failed
```

**原因：** 服务器无显示器，需要使用 offscreen 渲染。

```bash
# 在命令前添加
export MUJOCO_GL=osmesa   # 或 egl
export PYOPENGL_PLATFORM=osmesa

# 安装 osmesa
apt-get install -y libosmesa6-dev
```

## 2. 数据加载问题

### 2.1 训练时 OOM（内存不足）崩溃

**症状：** 进程在训练开始几分钟后被 kill，日志无明显 GPU OOM 信息。

**原因：** `shuffle_buffer_size` 过大，TensorFlow RLDS 加载器耗尽 CPU 内存。

```bash
# 必须添加此参数，且值不超过 2000
--shuffle_buffer_size 2000
```

### 2.2 `tf.errors.NotFoundError: dataset_info.json not found`

**原因：** 数据集路径不正确。

```bash
# 检查路径结构
ls tmp/datasets/rlds/libero_spatial/
# 应该看到：1.0.0/ 目录 和 dataset_info.json

# 确认参数
--data_root_dir tmp/datasets/rlds
--dataset_name  libero_spatial
# 注意：data_root_dir 是 rlds/ 这一层，不是 libero_spatial/ 这一层
```

### 2.3 `ValueError: Cannot find unnorm_key`

**原因：** 动作反归一化键与数据集不匹配。

```bash
# 方法 1：让系统自动检测（留空）
--unnorm_key ""
# 方法 2：查看数据集支持的 keys
python -c "
import tensorflow_datasets as tfds
b = tfds.builder_from_directory('tmp/datasets/rlds/libero_spatial')
print(b.info.metadata)
"
```

## 3. GPU 显存问题

### 3.1 训练时 `CUDA out of memory`

**诊断：**
```bash
# 查看当前显存使用
nvidia-smi
```

**解决方案（按顺序尝试）：**

```bash
# 1. 减小 batch size（默认 8，尝试 4 或 2）
--batch_size 4

# 2. 确认使用 LoRA（大幅减少可训练参数）
--use_lora True --lora_rank 32

# 3. 关闭图像增强
--image_aug False

# 4. 关闭 proprio（减少输入维度）
--use_proprio False

# 5. 减少输入图像数量（从 2 减到 1）
--num_images_in_input 1
```

### 3.2 评估时显存泄漏（长时间评估后 OOM）

**症状：** 前几个 episode 正常，后续 episode 越来越慢，最终 OOM。

**原因：** MuJoCo 渲染器或 PyTorch 推理时未释放显存。

```bash
# 方法 1：减少每次评估的 episode 数，分批运行
--num_trials_per_task 10  # 而不是 50

# 方法 2：在评估脚本中添加显存清理
# 在 run_capra_eval.py 的 episode 循环末尾添加：
import torch, gc
torch.cuda.empty_cache()
gc.collect()
```

## 4. 挖掘（Mining）问题

### 4.1 挖掘缓存为空，训练中 `activation_ratio=0`

**诊断：**
```bash
# 检查缓存目录
ls tmp/capra_cache/libero_spatial/

# 检查某个缓存文件中的激活步数
python -c "
import json, pathlib, glob
files = list(pathlib.Path('tmp/capra_cache/libero_spatial').glob('*.json'))
if not files:
    print('缓存目录为空！请先运行 mine_capra.sh')
else:
    d = json.loads(files[0].read_text())
    activated = sum(1 for r in d['records'] if len(r.get('equivalent_indices',[]))>0)
    print(f'文件: {files[0].name}, 总步数: {len(d["records"])}, 激活步数: {activated}')
"
```

**如果激活步数为 0：**
- 检查 `progress_floor` 是否过高（默认 0.20，尝试降低到 0.10）
- 检查 `epsilon_p_abs` 是否过小（默认 0.05，尝试增大到 0.10）
- 确认任务在执行时确实有进度（baseline 模型成功率需要 > 0）

### 4.2 挖掘速度极慢

**原因：** K 个 rollout 每步都要重置环境，开销大。

```bash
# 减少候选数和 rollout 步数（牺牲质量换速度）
--K 4 --H_s 3

# 只挖一个 episode 测试速度
--num_mining_episodes 1
```

## 5. 评估问题

### 5.1 `SPIR=0, EAR=0`（非 Baseline 模型）

**这不一定是错误。** 两种情况：

1. `capra_eval_K=0`（默认）：没有反事实 rollout，SPIR/EAR 必然为 0，这是正确的空值。
2. `activation_rate_mean=0`：E_t 在所有步骤都为空，检查 `epsilon_p_abs` 和 `progress_floor`。

```bash
# 要获得非零 SPIR/EAR，需要：
--capra_eval_K 8  # 启用 CF rollout
# 注意：需要服务器支持 MuJoCo sim.get_state()/set_state()
```

### 5.2 `AttributeError: 'OffScreenRenderEnv' object has no attribute 'sim'`

**原因：** LIBERO 版本不支持直接访问 `.sim`。

**临时解决方案：**
```bash
# 退回到 obs-only 模式（不使用 CF eval）
--capra_eval_K 0
```

### 5.3 评估结果目录为空

```bash
# 检查 log 目录
ls experiments/logs/capra/

# 如果目录不存在，确认路径参数
--local_log_dir experiments/logs/capra

# 查看评估日志中的错误
cat experiments/logs/capra/CAPRA-EVAL-*/eval_log.txt | tail -50
```

## 6. 冒烟测试失败

### 6.1 某个 check 报错

```bash
# 直接运行 Python 脚本查看详细错误
python scripts/capra/_smoke_logic.py
```

常见失败原因：

| Check | 常见原因 |
|---|---|
| [1] CAPRAConfig | `pip install -e .` 未执行 |
| [2] FinetuneCAPRAConfig | `vla-scripts/` 未在 Python 路径中 |
| [5] KL loss | PyTorch 版本不匹配 |
| [10] 程序化模板 | `procedural_splits.py` 导入失败 |

### 6.2 `UnicodeDecodeError` 或中文乱码

**原因：** Windows 系统默认编码为 GBK，Python 读取 UTF-8 文件报错。

```bash
# 设置环境变量
set PYTHONUTF8=1
# 或在 Python 命令前
python -X utf8 scripts/capra/_smoke_logic.py
```

## 7. 快速诊断清单

遇到问题时，按以下顺序检查：

```bash
# 1. 冒烟测试（最快验证逻辑层）
bash scripts/capra/smoke_capra.sh

# 2. 检查数据集路径
ls tmp/datasets/rlds/libero_spatial/

# 3. 检查模型路径
ls tmp/models/openvla-oft-libero/

# 4. 检查挖掘缓存（仅 CAPRA 训练需要）
ls tmp/capra_cache/libero_spatial/ | wc -l

# 5. 检查 GPU 状态
nvidia-smi

# 6. 检查 Python 环境
conda activate openvla
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```
