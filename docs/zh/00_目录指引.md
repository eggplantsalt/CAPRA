## 2. CAPRA 核心库详细目录

```
experiments/robot/capra/
│
├── core/                    ⚙️ 核心算法（算法理解从这里开始）
│   ├── capra_config.py      所有超参数的唯一入口（最常改的文件）
│   ├── capra_loss.py        CAPRA KL 散度损失函数实现
│   ├── equivalence.py       任务等价集 E_t 的过滤逻辑
│   ├── footprint.py         足迹函数 F_t(a) 三分量分解
│   ├── signals.py           从 obs dict / sim 提取状态信号
│   └── state_api.py         signals.py 的 re-export 兼容层
│
├── scene/                   🏠 场景语义（对象角色 + 进度 + 归因）
│   ├── object_roles.py      对象角色分类：TARGET/PROTECTED/NON_TARGET/IRRELEVANT
│   ├── task_progress.py     任务进度估计函数 P_t(a)
│   ├── precursor.py         前驱归因：PrecursorChain，R_t 计算
│   └── build_capra_dataset.py  q_hat_t 安全目标分布构建器
│
├── mining/                  ⛏️ 离线挖掘（生成训练监督信号）
│   ├── run_capra_mining.py  挖掘主循环入口（CLI 命令行入口）
│   ├── mining_cache.py      挖掘缓存的读写管理
│   ├── buffer.py            训练样本读取器（给 finetune_capra.py 用）
│   ├── rollout.py           短时 counterfactual rollout 执行
│   ├── candidate_actions.py 候选动作采样（nominal + 噪声扰动）
│   ├── snapshot.py          MuJoCo 状态快照/还原（CF rollout 基础）
│   └── env_adapter.py       LIBERO 环境的 CAPRA 适配器
│
├── eval/                    📊 评估与报告
│   ├── run_capra_eval.py    评估主循环入口（CLI 命令行入口）
│   ├── metrics.py           SPIR/EAR/EditGain/LeadTime 指标定义与计算
│   ├── report_utils.py      JSON/CSV/Markdown 结果报告生成
│   └── procedural_splits.py 四种程序化 side-effect 场景模板
│
└── *.py (shim 文件)          向后兼容导入层（内容仅一行 import *）
```

## 3. 你最可能要改的文件

| 场景 | 要改的文件 | 改什么 |
|---|---|---|
| 修改模型路径/数据集 | `core/capra_config.py` | `vla_path`, `dataset_name`, `data_root_dir` |
| 调整训练超参数 | `core/capra_config.py` | `lam`, `beta`, `rho`, `learning_rate` |
| 调整安全约束严格程度 | `core/capra_config.py` | `epsilon_p_abs`, `progress_floor`, `alpha_r` |
| 改变候选动作数量 | `core/capra_config.py` | `K`, `H_s` |
| 修改挖掘 episode 数 | `core/capra_config.py` | `num_mining_episodes` |
| 跑训练 | `vla-scripts/finetune_capra.py` | 一般不改，用 CLI 参数控制 |
| 跑评估 | `eval/run_capra_eval.py` | 一般不改，用 CLI 参数控制 |
| 添加新 side-effect 模板 | `eval/procedural_splits.py` | 参照现有 4 个模板格式 |
| 修改对象角色分配逻辑 | `scene/object_roles.py` | `assign_roles_from_task_description` |

## 4. 原版 openvla-oft 代码在哪里

| 组件 | 路径 | 说明 |
|---|---|---|
| Baseline 训练 | `vla-scripts/finetune.py` | 不要改，CAPRA 不修改此文件 |
| LIBERO 评估工具 | `experiments/robot/libero/` | CAPRA eval 复用了这里的大部分 helper |
| 动作头 | `prismatic/models/action_heads.py` | L1/Diffusion 动作头 |
| VLA 模型 | `prismatic/extern/hf/modeling_prismatic.py` | OpenVLAForActionPrediction |
| 训练工具 | `prismatic/training/train_utils.py` | loss helper, action mask |
| 常量 | `prismatic/vla/constants.py` | NUM_ACTIONS_CHUNK=8, ACTION_DIM=7 |

## 5. 脚本入口速查

```bash
# 1. 纯 Python 逻辑冒烟测试（不需要 GPU / LIBERO）
bash scripts/capra/smoke_capra.sh

# 2. 离线挖掘监督信号
bash scripts/capra/mine_capra.sh

# 3. CAPRA 训练
bash scripts/capra/train_capra.sh

# 4. 评估（CAPRA 或 Baseline 模型均可）
bash scripts/capra/eval_capra.sh
```

详细命令参数见 `docs/zh/03_训练指南.md` 和 `docs/zh/04_评估指南.md`。
