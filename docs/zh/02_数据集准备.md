# 02 数据集准备指南

> 本文说明 LIBERO RLDS 数据集的格式要求、存放规则和配置方法。

---

## 1. RLDS 数据集简介

CAPRA 使用 **RLDS（Reinforcement Learning Datasets）** 格式的数据集，
这是 Google 推广的一种标准化机器人轨迹存储格式，基于 TensorFlow Dataset。

每个 RLDS 数据集包含：
- 若干 **episode**（完整的机器人操作轨迹）
- 每个 episode 由若干 **step** 组成
- 每个 step 包含：观测图像、动作、语言指令、成功标志等

---

## 2. 目录结构要求

```
tmp/
└── datasets/
    └── rlds/
        ├── libero_spatial/        ← 对应 --dataset_name libero_spatial
        │   ├── 1.0.0/
        │   │   ├── dataset_info.json
        │   │   ├── features.json
        │   │   └── *.tfrecord-*   ← 实际数据分片
        │   └── dataset_info.json
        ├── libero_object/
        ├── libero_goal/
        ├── libero_10/
        └── libero_90/
```

**关键规则：**
- `data_root_dir` 必须指向 `tmp/datasets/rlds/`（包含各数据集子目录的那一层）
- 子目录名称必须与 `--dataset_name` 参数完全一致
- 数据集内部必须包含 `dataset_info.json` 和 `.tfrecord` 文件

## 3. 下载命令

```bash
conda activate openvla
mkdir -p tmp/datasets/rlds

# 单个数据集（推荐先只下 libero_spatial 验证流程）
huggingface-cli download openvla/libero-spatial-no-noops-rlds \
    --local-dir tmp/datasets/rlds/libero_spatial

# 全部 5 个 LIBERO 数据集
for SUITE in libero_spatial libero_object libero_goal libero_10 libero_90; do
    huggingface-cli download openvla/${SUITE}-no-noops-rlds \
        --local-dir tmp/datasets/rlds/${SUITE}
    echo "Downloaded: ${SUITE}"
done
```

## 4. RLDS 必要字段说明

CAPRA 训练脚本从 RLDS 数据集中读取以下字段：

| 字段路径 | 类型 | 说明 |
|---|---|---|
| `steps/observation/image` | `uint8 (H,W,3)` | 第三人称摄像头图像 256x256 |
| `steps/observation/wrist_image` | `uint8 (H,W,3)` | 手腕摄像头图像（如有） |
| `steps/observation/state` | `float32 (8,)` | 本体感知状态（关节角 + 夹爪） |
| `steps/action` | `float32 (7,)` | 动作向量（末端位姿 delta + 夹爪） |
| `steps/language_instruction` | `string` | 语言指令，如 "pick the mug" |
| `steps/is_last` | `bool` | 是否为 episode 最后一步 |

**注意：** `no-noops` 版本已过滤掉无动作步骤（`action ≈ 0`），推荐使用。

## 5. 配置训练脚本中的路径

### 方式一：通过 CLI 参数（推荐）

```bash
torchrun --standalone --nnodes 1 --nproc-per-node 1 \
    vla-scripts/finetune_capra.py \
    --vla_path          tmp/models/openvla-oft-libero \
    --dataset_name      libero_spatial \
    --data_root_dir     tmp/datasets/rlds \
    --run_root_dir      runs \
    --capra_enabled     True
```

### 方式二：修改默认值

编辑 `experiments/robot/capra/core/capra_config.py`：

```python
@dataclass
class CAPRAConfig:
    vla_path: str       = "tmp/models/openvla-oft-libero"  # ← 改这里
    dataset_name: str   = "libero_spatial"                  # ← 改这里
    data_root_dir: Path = Path("tmp/datasets/rlds")         # ← 改这里
```

## 6. 验证数据集是否可以被正确加载

```bash
conda activate openvla
cd /path/to/openvla-oft

# 用 Python 快速验证 RLDS 能被加载
python - <<'EOF'
import tensorflow as tf
import tensorflow_datasets as tfds

builder = tfds.builder_from_directory(
    builder_dir='tmp/datasets/rlds/libero_spatial'
)
ds = builder.as_dataset(split='train', shuffle_files=False)
for ep in ds.take(1):
    steps = list(ep['steps'].as_numpy_iterator())
    print(f'Episode steps: {len(steps)}')
    print(f'Action shape: {steps[0]["action"].shape}')
    print(f'Image shape: {steps[0]["observation"]["image"].shape}')
    print(f'Language: {steps[0]["language_instruction"]}')
    print('RLDS dataset OK')
EOF
```

**预期输出：**
```
Episode steps: 约200步
Action shape: (7,)
Image shape: (256, 256, 3)
Language: b'pick the alphabet soup ...'
RLDS dataset OK
```

## 7. 挖掘缓存（mining cache）的存储规则

挖掘产出的监督信号缓存结构：

```
tmp/capra_cache/
└── libero_spatial/           ← dataset_name 子目录
    ├── episode_0000.json     ← 每个 episode 一个 JSON 文件
    ├── episode_0001.json
    └── ...
```

每个 JSON 文件包含该 episode 所有时间步的：
- K 个候选动作及其 `P_t`, `F_t`
- 等价集 `E_t` 的索引
- 安全目标分布 `q_hat_t`
- 权重 `w_t`

**断点续传：** 已存在的文件会被自动跳过，中途中断后可以直接重跑。
