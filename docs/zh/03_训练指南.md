# 03 训练指南

> 本文覆盖：Baseline 训练、CAPRA 离线挖掘、CAPRA 训练、关键参数解读、WandB 指标说明。

---

## 1. 整体流程概览

```
步骤 1  [可选] 挖掘监督信号
          bash scripts/capra/mine_capra.sh
          输出 → tmp/capra_cache/libero_spatial/*.json

步骤 2a  Baseline 训练（对照组）
          bash scripts/capra/train_capra.sh  (capra_enabled=False)
          输出 → runs/CAPRA-libero_spatial-{datetime}/

步骤 2b  CAPRA 训练（实验组，需要先完成步骤 1）
          bash scripts/capra/train_capra.sh  (capra_enabled=True)
          输出 → runs/CAPRA-libero_spatial-{datetime}/
```

---

## 2. 步骤 1：离线挖掘监督信号

### 一键运行

```bash
conda activate openvla
cd /path/to/openvla-oft

bash scripts/capra/mine_capra.sh \
    tmp/models/openvla-oft-libero \
    libero_spatial \
    tmp/capra_cache
```

### 完整 CLI 参数

```bash
python -m experiments.robot.capra.mining.run_capra_mining \
    --pretrained_checkpoint tmp/models/openvla-oft-libero \
    --dataset_name          libero_spatial \
    --cache_root            tmp/capra_cache \
    --num_mining_episodes   50 \
    --K                     8 \
    --H_s                   5 \
    --candidate_noise_sigma 0.02 \
    --progress_floor        0.20 \
    --epsilon_p_abs         0.05 \
    --epsilon_p_rel         0.10 \
    --seed                  7
```

### 常用调整

| 想要... | 修改参数 | 说明 |
|---|---|---|
| 更快完成挖掘（低质量） | `--K 4 --H_s 3` | 减少候选和 rollout 步数 |
| 更高质量监督信号 | `--K 16 --H_s 8` | 更多候选，更准确的 P_t/F_t |
| 先快速验证流程 | `--num_mining_episodes 1` | 只挖一个 episode |
| 强制重新挖掘 | `--force_remine True` | 覆盖已有缓存 |

### 挖掘完成后验证

```bash
# 检查缓存文件是否产出
ls tmp/capra_cache/libero_spatial/ | head -10
# 预期：episode_0000.json, episode_0001.json ...

# 查看一个缓存文件的内容
python -c "
import json, pathlib
f = next(pathlib.Path('tmp/capra_cache/libero_spatial').glob('*.json'))
d = json.loads(f.read_text())
print('steps:', len(d['records']))
print('K:', len(d['records'][0]['candidate_actions']))
print('activated steps:', sum(1 for r in d['records'] if len(r['equivalent_indices'])>0))
"
```

## 3. 步骤 2a：Baseline 训练

Baseline 模式与原版 `finetune.py` 完全等价，但使用 `finetune_capra.py` 入口，
方便后续切换到 CAPRA 模式进行公平对比。

```bash
conda activate openvla
cd /path/to/openvla-oft

torchrun --standalone --nnodes 1 --nproc-per-node 1 \
    vla-scripts/finetune_capra.py \
    --vla_path              tmp/models/openvla-oft-libero \
    --dataset_name          libero_spatial \
    --data_root_dir         tmp/datasets/rlds \
    --run_root_dir          runs \
    --capra_enabled         False \
    --batch_size            8 \
    --learning_rate         5e-4 \
    --max_steps             200000 \
    --num_steps_before_decay 100000 \
    --save_freq             10000 \
    --shuffle_buffer_size   2000 \
    --use_l1_regression     True \
    --use_diffusion         False \
    --use_proprio           True \
    --num_images_in_input   2 \
    --use_lora              True \
    --lora_rank             32 \
    --center_crop           True \
    --image_aug             True
```

## 4. 步骤 2b：CAPRA 训练

**前提：** 步骤 1 的挖掘缓存必须已经存在于 `tmp/capra_cache/libero_spatial/`。

```bash
conda activate openvla
cd /path/to/openvla-oft

torchrun --standalone --nnodes 1 --nproc-per-node 1 \
    vla-scripts/finetune_capra.py \

    --capra_enabled         True \
    --lam                   0.1 \
    --rho                   0.5 \
    --beta                  1.0 \
    --capra_warmup_steps    500 \
    --batch_size            8 \
    --learning_rate         5e-4 \
    --max_steps             200000 \
    --shuffle_buffer_size   2000 \
    --use_l1_regression     True \
    --use_proprio           True \
    --num_images_in_input   2 \
    --use_lora              True \
    --lora_rank             32
```

## 5. 关键参数解读

### CAPRA 核心参数

| 参数 | 默认值 | 物理含义 | 调整建议 |
|---|---|---|---|
| `--lam` | 0.1 | CAPRA 损失权重：`L = L_anchor + lam * L_capra` | 从 0.1 开始；如果任务成功率下降，减小到 0.05 |
| `--beta` | 1.0 | 安全目标分布温度：越大越集中在最安全动作 | 1.0 是默认起点；0.5 更平滑，2.0 更激进 |
| `--rho` | 0.5 | 前驱归因上权：越大越鼓励早期预防 | 0 关闭前驱上权；0.5 推荐起点 |
| `--capra_warmup_steps` | 500 | 预热步数：此步数内只跑 anchor loss | 建议 300-1000，让模型先稳定 |
| `--epsilon_p_abs` | 0.05 | 等价集绝对进度差阈值 | 减小=更严格等价；增大=更多候选进入 E_t |
| `--progress_floor` | 0.20 | 触发 CAPRA 损失的最低进度 | 减小=更早触发；建议 0.15-0.25 |

### 必须注意的参数

| 参数 | 要求 | 原因 |
|---|---|---|
| `--shuffle_buffer_size` | **必须 ≤ 2000** | 更大的值会导致 TF RLDS 加载器 OOM 崩溃 |
| `--use_l1_regression` | 建议 True | 与 CAPRA 候选集近似兼容最好 |
| `--use_diffusion` | 建议 False | Diffusion head 与 CAPRA 评分近似不兼容 |

## 6. WandB 指标解读

启用 WandB：在命令中添加 `--use_wandb True --wandb_entity your-name`

| 指标名 | 含义 | 健康值参考 |
|---|---|---|
| `loss_value` | 总损失 = anchor + lam*capra | 持续下降为正常 |
| `anchor_loss` | L1 回归任务损失（模仿学习） | 应该稳定下降 |
| `capra_loss` | CAPRA KL 散度项 | 初期可能上升再下降 |
| `activation_ratio` | 本 batch 被 CAPRA 激活的样本比例 | 正常范围 0.1 - 0.5 |
| `mean_w_t` | 激活样本的平均损失权重 w_t | 越高=当前 batch 可避免风险越大 |
| `mean_delta_t` | 激活样本的平均可避免风险 Delta_t | 训练过程中应逐渐下降 |
| `lr` | 当前学习率 | MultiStepLR 在 num_steps_before_decay 后衰减 |

**异常信号：**
- `activation_ratio` 长期为 0：挖掘缓存为空或路径错误
- `capra_loss` 发散（持续上升超过 anchor_loss）：减小 `--lam`
- `anchor_loss` 不收敛：检查数据集路径和 `shuffle_buffer_size`

## 7. 训练输出说明

```
runs/
└── CAPRA-libero_spatial-{datetime}/
    ├── checkpoints/
    │   ├── step-010000/        ← 每 save_freq 步保存一次
    │   ├── step-020000/
    │   └── latest -> step-200000/  ← 最新检查点的符号链接
    └── metrics/
        └── train_metrics.jsonl ← 逐步训练指标
```

评估时使用 `runs/CAPRA-.../checkpoints/latest` 作为检查点路径。
