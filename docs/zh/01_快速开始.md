# 01 从零开始快速上手 (Quickstart)

> 目标：从一台干净的服务器出发，用最短路径验证环境可用，并跑通冒烟测试。

---

## 第一步：克隆代码仓库

```bash
git clone https://github.com/your-org/openvla-oft.git
cd openvla-oft
```

---

## 第二步：创建并激活 Conda 环境

```bash
conda create -n openvla python=3.10 -y
conda activate openvla

# 根据你的 CUDA 版本选择（CUDA 11.8）：
pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu118
# CUDA 12.1：
# pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu121
```

---

## 第三步：安装项目依赖

```bash
# 安装 prismatic + openvla-oft 依赖
pip install -e .

# 安装 LIBERO 仿真环境
pip install libero

# 安装其他依赖
pip install draccus wandb imageio tqdm accelerate peft
pip install tensorflow        # 用于 RLDS 数据加载
```

---

## 第四步：验证纯 Python 逻辑（无需 GPU，无需模型）

```bash
conda activate openvla
cd /path/to/openvla-oft
bash scripts/capra/smoke_capra.sh
```

**预期输出（最后一行）：**
```
[smoke_capra] All checks passed.
```

如果冒烟测试通过，说明所有 CAPRA Python 模块都可以正确导入和运行。

---

## 第五步：下载预训练模型

```bash
mkdir -p tmp/models

# 下载 OpenVLA-OFT 在 LIBERO 上的基线检查点
huggingface-cli login   # 输入你的 HuggingFace token

huggingface-cli download openvla/openvla-oft-libero \
    --local-dir tmp/models/openvla-oft-libero
```

**路径要求：** 模型必须放在 `tmp/models/openvla-oft-libero/`，
否则需要修改 `core/capra_config.py` 中的 `pretrained_checkpoint` 和 `vla_path`。

---

## 第六步：下载数据集

```bash
mkdir -p tmp/datasets/rlds

# 以 libero_spatial 为例
huggingface-cli download openvla/libero-spatial-no-noops-rlds \
    --local-dir tmp/datasets/rlds/libero_spatial
```

全部 LIBERO 数据集：
```bash
for SUITE in libero_spatial libero_object libero_goal libero_10 libero_90; do
    huggingface-cli download openvla/${SUITE}-no-noops-rlds \
        --local-dir tmp/datasets/rlds/${SUITE}
done
```

---

## 第七步：验证 Baseline 评估能跑通

```bash
conda activate openvla
cd /path/to/openvla-oft

# 用 baseline 模型跑 3 个 episode 快速验证环境
python -m experiments.robot.capra.eval.run_capra_eval \
    --pretrained_checkpoint tmp/models/openvla-oft-libero \
    --task_suite_name libero_spatial \
    --num_trials_per_task 3 \
    --capra_eval_K 0
```

**预期：** 在 `experiments/logs/capra/CAPRA-EVAL-libero_spatial-{datetime}/` 下
生成 `results_aggregate.json`，其中 `success_rate` 字段有合理数值（不为 null）。

## 常见安装问题

| 问题 | 原因 | 解决方案 |
|---|---|---|
| `ModuleNotFoundError: libero` | LIBERO 未安装 | `pip install libero` |
| `ModuleNotFoundError: draccus` | draccus 未安装 | `pip install draccus` |
| `ImportError: cannot import name 'CAPRAConfig'` | 未安装项目 | `pip install -e .` |
| TF 加载数据集报 OOM | shuffle_buffer_size 过大 | 确认 `--shuffle_buffer_size 2000` |
| CUDA out of memory（训练时） | batch_size 过大或模型太大 | 减小 `--batch_size`，启用 `--use_lora True` |
| `MuJoCo error` | mujoco-py 未正确安装 | 参考 LIBERO 官方安装指引 |

详细故障排查见 `docs/zh/06_故障排查.md`。
